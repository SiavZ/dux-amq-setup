     happened in 6.2). amq wake's TTY injection therefore works on this kernel.

Strace evidence (from artifact 07-tiocsti-strace.txt):
  See docs/plans/audits/audit01/artifacts/07-tiocsti-strace.txt
  Phase 07 verdict: amq wake **DOES** call ioctl(_, TIOCSTI, ...), one
  ioctl per character of the injected payload (110 calls observed for
  a 110-char message). It does NOT use PTY-master writes. On this
  Debian 12 / kernel 6.1 host TIOCSTI is unrestricted (the
  legacy_tiocsti sysctl was added in 6.2 and defaults to 0 there), so
  injection works correctly here. On a kernel 6.2+ host (Ubuntu 24.04
  default, Debian 12-with-backports), the injection silently fails —
  that is the audit P1-1 hazard. It is mitigated by README "Kernel
  compatibility" (sysctl pin) and the `amq wake --inject-via <bin>`
  external-injector option for non-root environments.
