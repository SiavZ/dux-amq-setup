=== Verdict: Ubuntu 24.04 user-space â€” PASS (kernel-level coverage GAP, see note) ===
  - Ubuntu 24.04 apt set provides curl/jq/tar/git/rsync; install.sh preflight passes.
  - install.sh completes: dux + amq + skills + wrappers + lib all installed.
  - dux-amq-doctor reports overlay 0.1.0 / amq 0.34.0 in the container.

  Coverage GAP: kernel 6.2+ behaviour (legacy_tiocsti sysctl present and
  defaulting to 0) cannot be exercised in Docker because Docker uses the
  host kernel. To close this gap on a real 6.2+ Ubuntu 24.04 VM, run:

    cat /proc/sys/dev/tty/legacy_tiocsti          # expect: 0
    sudo sysctl -w dev.tty.legacy_tiocsti=1       # workaround if needed
    amq wake --me <id> --root /data/state/amq --inject-mode raw </dev/tty &
    strace -p $! -e ioctl 2>&1 | head -20         # confirm transport

  Phase 07 verified amq 0.34.0 **DOES** call TIOCSTI (one ioctl per
  injected character) on this kernel. See 07-tiocsti-result.md and the
  raw artifact 07-tiocsti-strace.txt. The audit P1-1 risk is therefore
  real on 6.2+ kernels and is mitigated by the README "Kernel
  compatibility" workaround (sysctl pin or --inject-via external
  injector), not by upstream amq itself. Filing an upstream patch to
  switch amq to posix_openpt(3) is recommended follow-up but out of
  scope for this audit cycle.
